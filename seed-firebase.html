Regras de segurança do Firestore

Copie o bloco abaixo e cole em Firestore Database → Regras no Firebase Console:

rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {

    function isSignedIn() { return request.auth != null; }
    function isAdmin() {
      return isSignedIn() &&
             exists(/databases/$(database)/documents/usuarios/$(request.auth.uid)) &&
             get(/databases/$(database)/documents/usuarios/$(request.auth.uid)).data.perfil == 'admin';
    }

    // Usuários: próprio documento (leitura) + admin (tudo)
    match /usuarios/{uid} {
      allow read: if isSignedIn() && (request.auth.uid == uid || isAdmin());
      allow write: if isAdmin();
    }

    // Perfis: qualquer logado lê, só admin escreve
    match /perfisTemplate/{sigla} {
      allow read: if isSignedIn();
      allow write: if isAdmin();
    }

    // Auditoria: qualquer logado escreve (append-only), só admin lê
    match /auditLog/{logId} {
      allow create: if isSignedIn();
      allow read: if isAdmin();
      allow update, delete: if false;
    }

    // Configurações por base: qualquer logado lê, só admin escreve
    match /configuracoes/{base} {
      allow read: if isSignedIn();
      allow write: if isAdmin();
    }
  }
}
Importante sobre a ordem:
Se você publicar essas regras antes de rodar o seed, o seed vai falhar porque a coleção usuarios ainda não tem seu admin (a regra isAdmin() depende disso).

Solução: Publique as regras temporárias abaixo primeiro, rode o seed, e depois troque pelas regras finais.
Regras temporárias (só para rodar o seed) 
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /{document=**} {
      // TEMPORÁRIO: permite qualquer usuário logado ler/escrever.
      // Trocar pelas regras de produção APÓS rodar o seed.
      allow read, write: if request.auth != null;
    }
  }
}
